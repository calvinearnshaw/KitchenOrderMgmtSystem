KOMS

A Node.JS project by Calvin Earnshaw


Introduction

This is a small Node.JS project I worked on during early Summer 2023. You may find the following information useful whilst navigating this project.

This Kitchen Management System simulates a typical fast food restaurant, complete with a customer-facing kiosk ordering system, a customer-facing order tracker, and kitchen order monitor.

Please note this project remains a work in progress, and should be treated as such. The project uses the Socket.IO library to communicate information between each of the components of the system, therefore each tab will not save data if refreshed.


How to Use

This guide assumes you have successfully started the Node.JS server, and have accessed this webpage through your preferred web browser, using the address https://localhost:6500. Please note, however, that Google Chrome is recommended.

Select any of the three menu items above to access the various areas of the system. These will open in separate tabs.


Kitchen Order Monitor

To be displayed in the kitchen, contains a live feed of orders that are processed by the kiosk. Each order has its own timer, showing how long the order has been waiting to be processed by the kitchen, as well as a visual red/green indicator to visually indicate the time taken to process an order.

The Space Bar on your keyboard acts as a 'serve' button. The typical use case is to press this after an order's been made by the kitchen, and is ready to be served to the customer.

Pressing the 'V' key will alter the zoom level of the Kitchen Order Monitor, allowing for easier reading of the information on-screen, especially if the monitor were to be mounted further away from the user.

Pressing the 'C' key will open a small popup window used for order intake testing, and won't serve part of the final product.

Along the bottom of the Monitor, there is a live clock, an indicator of both the average and maximum order processing times recorded since the Monitor was accessed, and an indicator of pending orders, which will indicate white whenever there are pending orders that cannot be immediately displayed on the screen.


Customer Order Monitor

This provides a 'live' feed to customers relating to the progress of their order. Currently only has two main sections - 'Preparing' and 'Please Collect', along with an order number.

This has no interactive parts - the order number will move automatically as soon as the serve button is hit on the Kitchen Order Monitor. Once an order's been served, the customer order monitor will clear the order number after 60 seconds.


Customer Kiosk

An interactive ordering system for customers to place their orders, styled similarly to that of typical high street fast food chains. Allows products to be selected, quantities updated, and orders to be processed.

This is currently a work in progress, with more features being introduced in due course, such as:

  • Order Customisations
  • Food Categories (these buttons currently do not work)
